# .gitignore 指南

本文档解释 NanoMQ 项目的 `.gitignore` 规则及其设计原则。

## 设计原则

1. **仅跟踪源代码** - 构建产物、依赖、缓存不应进入仓库
2. **多配置支持** - 支持不同的构建目录（debug、release、deb、taos 等）
3. **跨平台** - 忽略 Windows、macOS、Linux 特定的文件
4. **清晰分类** - 按类型组织规则便于维护和理解

## 规则详解

### 编译对象 & 可执行文件

```gitignore
*.o *.ko *.obj *.elf    # 目标文件
*.dll *.so *.dylib      # 共享库
*.exe *.out *.app       # 可执行文件
```

**为什么忽略**: 这些是编译过程的中间产物或最终产物，随处可重新生成，不应提交。

---

### CMake 构建产物

```gitignore
build/                      # 默认构建目录
build-deb/                  # DEB 包构建
build-deb-taos/             # TDengine 集成构建
build-tests/                # 测试构建
_CPack_Packages/            # 打包工件
CMakeFiles/ CMakeCache.txt  # CMake 生成文件
```

**为什么忽略**: 每个开发者和 CI 环境都会生成自己的构建目录。为避免合并冲突：
- 使用 `-DCMAKE_BUILD_TYPE=Release` 等构建选项
- 不同场景使用不同构建目录（开发用 `build/`，打包用 `build-deb-*`)

---

### 依赖

```gitignore
NanoNNG/            # NanoNNG 子仓库（可选）
extern/l8w8jwt/     # JWT 库（本地克隆时）
```

**为什么忽略**:
- 这些可以通过 `git submodule update --init` 或脚本恢复
- 避免嵌套仓库变更干扰主项目

---

### IDE & 编辑器

```gitignore
.idea/              # JetBrains IDEs (CLion, IntelliJ)
.vscode/            # Visual Studio Code
.claude/            # Claude Code 本地上下文
*.swp *.swo         # Vim 交换文件
```

**为什么忽略**:
- 这些是个人开发环境配置
- 不同开发者可能使用不同 IDE
- 包含用户敏感信息（调试器路径、敏感设置）

**建议**: 如需共享 IDE 配置，创建 `.vscode-template/` 或 `docs/ide-setup/` 目录

---

### 操作系统文件

```gitignore
# macOS
.DS_Store .AppleDouble

# Windows  
Thumbs.db desktop.ini

# Linux
.directory .Trash-*
```

**为什么忽略**: OS 自动生成的元数据文件，用户无法控制。

---

## 处理特殊场景

### 需要提交但想本地忽略的文件

使用 **`.git/info/exclude`** 而不是 `.gitignore`：

```bash
# 仅影响本地仓库，不影响其他开发者
echo "my-local-build/" >> .git/info/exclude
```

---

### 添加新的构建变体

如果添加新的构建目录（如 `build-clang/`）：

```bash
# 在 .gitignore 的"Build & CMake Artifacts"部分添加
build-clang/

# 创建提交
git add .gitignore
git commit -m "docs: add build-clang to .gitignore"
```

---

### 防止意外提交大文件

设置 Git 钩子检查：

```bash
#!/bin/bash
# .git/hooks/pre-commit
if git diff --cached --name-only | xargs du -b | awk '$1 > 100000000 { exit 1 }'; then
    exit 0
else
    echo "Error: Attempting to commit files >100MB"
    exit 1
fi
```

---

## 验证规则有效性

检查是否有应该被忽略但未被忽略的文件：

```bash
# 1. 列出所有未追踪文件
git status --short | grep "^??"

# 2. 检查是否有应被忽略的文件被意外追踪
git check-ignore -v <file>

# 3. 模拟 .gitignore 规则（不实际变更）
git clean -fdXn

# 4. 移除已追踪但应被忽略的文件
git rm --cached <file>
git commit -m "docs: remove <file> from tracking (should be in .gitignore)"
```

## 相关文件

| 文件 | 用途 |
|------|------|
| `.gitignore` | **项目级** - 所有开发者都遵守 |
| `.git/info/exclude` | **个人级** - 仅当前开发者 |
| `.gitattributes` | **跨平台规范** - 行结束符、二进制标记 |
