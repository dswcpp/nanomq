# 焊接遥测入库上线说明

本文档用于运维上线 NanoMQ 焊接遥测解析入库能力。

针对全新 Ubuntu 主机，优先使用以下两份文档：

- [01-Ubuntu全新主机-NanoMQ安装命令.md](/home/tery/project/nanomq/public/0.24.14/01-Ubuntu%E5%85%A8%E6%96%B0%E4%B8%BB%E6%9C%BA-NanoMQ%E5%AE%89%E8%A3%85%E5%91%BD%E4%BB%A4.md)
- [02-Ubuntu全新主机-TDengine安装与初始化.md](/home/tery/project/nanomq/public/0.24.14/02-Ubuntu%E5%85%A8%E6%96%B0%E4%B8%BB%E6%9C%BA-TDengine%E5%AE%89%E8%A3%85%E4%B8%8E%E5%88%9D%E5%A7%8B%E5%8C%96.md)

## 1. 交付物

- 安装包: [nanomq-nng-v0.24.14.deb](/home/tery/project/nanomq/public/0.24.14/nanomq-nng-v0.24.14.deb)
- MQTT 规则配置: [nanomq_weld_taos.conf](/home/tery/project/nanomq/public/0.24.14/nanomq_weld_taos.conf)
- NanoMQ 密码文件: [nanomq_pwd.conf](/home/tery/project/nanomq/public/0.24.14/nanomq_pwd.conf)
- TDengine 建库建表脚本: [weld_tdengine_schema.sql](/home/tery/project/nanomq/public/0.24.14/weld_tdengine_schema.sql)
- TDengine 重建脚本: [weld_tdengine_schema_reset.sql](/home/tery/project/nanomq/public/0.24.14/weld_tdengine_schema_reset.sql)
- 校验命令: `sha256sum nanomq-nng-v0.24.14.deb`

本次交付包含以下变更：

- 高频 `telemetry/power` 改为按消息写入 `weld_current_raw` / `weld_voltage_raw`
- NanoMQ 不再逐点展开 `uniform_series_binary`
- `nanomq_cli pub` 成功发送后返回 `0`
- 新增 `pub_handler weld_taos`、`weld_telemetry`、`weld_taos_sink` 三段统计日志

## 2. 安装

```bash
cd /path/to/0.24.14
sha256sum ./nanomq-nng-v0.24.14.deb
sudo dpkg -i ./nanomq-nng-v0.24.14.deb
```

安装后默认文件位置：

- `/usr/bin/nanomq`
- `/usr/local/etc/nanomq_weld_taos.conf`
- `/usr/local/etc/weld_tdengine_schema.sql`

## 3. 上线前检查

确认以下条件：

- TDengine REST 服务可访问，默认端口 `6041`
- NanoMQ 监听端口 `1883` 未被占用
- NanoMQ HTTP 管理端口 `8083` 未被占用
- TDengine 用户对目标库有建库建表和写入权限

检查命令：

```bash
ss -ltnp | rg ':1883|:8083|:6041'
```

## 4. TDengine 初始化

全新环境执行建库建表脚本：

```bash
taos < /usr/local/etc/weld_tdengine_schema.sql
```

如果 `mqtt_rule` 已经存在，必须先确认其精度是 `us`：

```sql
SHOW CREATE DATABASE mqtt_rule;
```

若不是 `PRECISION 'us'`，不要继续使用旧库。处理方式二选一：

1. 删除旧库后，重新执行同一个初始化脚本

```bash
taos -s "DROP DATABASE IF EXISTS mqtt_rule"
taos < /usr/local/etc/weld_tdengine_schema.sql
```

2. 修改 NanoMQ 配置中的 `database`，换一个全新的库名，再按初始化脚本创建

脚本会创建：

- 数据库 `mqtt_rule`
- 超表 `weld_env_point`
- 超表 `weld_flow_point`
- 超表 `weld_current_raw`
- 超表 `weld_voltage_raw`

> 电流/电压的 raw 表不再展开 `uniform_series_binary`，NanoMQ 原样保存 `data.payload` 并记录 `window_start_us`/`sample_rate_hz`/`point_count`，下游需按这些字段恢复每个采样点的时间。

## 5. 配置

本次上线使用：

- 配置文件 `/usr/local/etc/nanomq_weld_taos.conf`
- 数据库 `mqtt_rule`
- TDengine 地址 `127.0.0.1:6041`

如果现场账号、密码、地址不同，只修改以下配置项：

- `host`
- `port`
- `username`
- `password`
- `database`

如果现场写库吞吐高、TDengine 存在瞬时抖动，可按需调整焊接解析异步队列：

- `NANOMQ_WELD_PARSE_WORKER_MAX`
- `NANOMQ_WELD_PARSE_QUEUE_MAX_MSG`
- `NANOMQ_WELD_PARSE_QUEUE_MAX_BYTES`

默认值当前为：

- worker `8`
- queue message `2048`
- queue bytes `268435456`

上线后建议观察以下三段统计日志：

- `pub_handler weld_taos`
- `weld_telemetry`
- `weld_taos_sink`

## 6. 启动

```bash
nanomq start --conf /usr/local/etc/nanomq_weld_taos.conf --log_level info
```

如需停止：

```bash
nanomq stop
```

## 7. 规则说明

四类 MQTT 遥测规则如下：

- `weld/+/+/+/+/+/telemetry/env/#` -> `weld_env_point`
- `weld/+/+/+/+/+/telemetry/flow/#` -> `weld_flow_point`
- `weld/+/+/+/+/+/telemetry/power/#` 且 `payload.signal_type = 'current'` -> `weld_current_raw`
- `weld/+/+/+/+/+/telemetry/power/#` 且 `payload.signal_type = 'voltage'` -> `weld_voltage_raw`

## 8. 上线后验证

检查 NanoMQ 进程：

```bash
ps -ef | rg 'nanomq start'
```

检查端口：

```bash
ss -ltnp | rg ':1883|:8083'
```

检查 TDengine 表：

```sql
USE mqtt_rule;
SHOW STABLES;
DESCRIBE weld_env_point;
DESCRIBE weld_flow_point;
DESCRIBE weld_current_raw;
DESCRIBE weld_voltage_raw;
```

公共列里新增了 `task_id`，用于承接焊接消息公共头任务标识；旧消息该列允许为 `NULL`，当前阶段默认占位符常见为 `"110120119"`。

如果目标环境的 `mqtt_rule` 已经是旧版表结构，需要先补列或重建后再升级 NanoMQ：

```sql
ALTER STABLE mqtt_rule.weld_env_point ADD COLUMN task_id NCHAR(128);
ALTER STABLE mqtt_rule.weld_flow_point ADD COLUMN task_id NCHAR(128);
ALTER STABLE mqtt_rule.weld_current_raw ADD COLUMN task_id VARCHAR(128);
ALTER STABLE mqtt_rule.weld_voltage_raw ADD COLUMN task_id VARCHAR(128);
```

也可以直接执行 `etc/weld_tdengine_schema_reset.sql` 重新建库。

检查是否已产生子表：

```sql
USE mqtt_rule;
SHOW TABLES;
```

检查是否已有写入：

```sql
SELECT COUNT(*) FROM weld_env_point;
SELECT COUNT(*) FROM weld_flow_point;
SELECT COUNT(*) FROM weld_current_raw;
SELECT COUNT(*) FROM weld_voltage_raw;
SELECT ts, topic_name, task_id, point_count, encoding FROM weld_current_raw ORDER BY ts DESC LIMIT 5;
```

## 9. 异常排查

若 NanoMQ 启动失败，优先检查：

- `1883` 端口已被旧 NanoMQ 或其他 MQTT Broker 占用
- 配置文件路径错误
- TDengine `6041` 未启动或不可访问

若 MQTT 已接收但未写库，优先检查：

- `payload.signal_type` 是否与规则匹配
- TDengine 账号密码是否正确
- 目标数据库是否为 `mqtt_rule`
- `mqtt_rule` 是否为 `PRECISION 'us'`
- 报文是否符合焊接遥测规范
- NanoMQ 日志中是否出现 `weld_telemetry_async: queue pressure`
- NanoMQ 日志中是否出现 `weld_telemetry_async: queue full, drop message`
- NanoMQ 日志中 `pub_handler weld_taos` 的 `rule_match` / `enqueue_ok` 是否增长
- NanoMQ 日志中 `weld_telemetry` 的 `current_msg` / `voltage_msg` 是否增长
- NanoMQ 日志中 `weld_taos_sink` 的 `enqueue_msg` / `row_ok` 是否增长

如果出现异步队列压力或丢弃日志，优先处理：

1. 提升 `NANOMQ_WELD_PARSE_QUEUE_MAX_MSG`
2. 提升 `NANOMQ_WELD_PARSE_QUEUE_MAX_BYTES`
3. 评估 `NANOMQ_WELD_PARSE_WORKER_MAX`
4. 确认 TDengine REST 和写入侧没有性能瓶颈

## 10. 回滚

停止新版本：

```bash
nanomq stop
```

切回旧配置或旧二进制后重新启动。

如果只需要临时关闭焊接入库，可直接停用：

- `/usr/local/etc/nanomq_weld_taos.conf`

并切回原有 NanoMQ 配置文件启动。
